# Payload Copy Hotspot Inventory

## Scope

This inventory tracks request, response, error, debug/snapshot, retry, continuation, provider-response effect, registry, and JS/Rust boundary payload copies. It is an audit and execution surface for reducing memory residency without changing true client/provider payload semantics.

## Classes

1. Boundary serialization: JS object -> JSON string -> Rust `serde_json::Value` -> JSON string -> JS object. This remains unavoidable while the N-API contract is JSON-string based.
2. Normal-path removable copies: duplicated full payload ownership inside request/response pipeline stages, effect plans, registries, and retry/continuation materialization.
3. Debug/error/snapshot-only copies: observability payload capture, errorsamples, contract observations, and stage traces. These may be budgeted or summarized, but live payload truth must not be trimmed.

## Verification Rules

- Request-path changes run `npm run verify:request-payload-copy-budget`, target `rustfmt --check`, and target `git diff --check`.
- Response effect changes run focused Rust/Jest provider-response gates, `tests/sharedmodule/hub-pipeline-stage-residue-audit.spec.ts`, `npm run build:native-hotpath`, and target `git diff --check`.
- Registry changes run responses registry focused tests, relevant response outbound semantics tests, `npm run build:native-hotpath`, and target `rustfmt --check`.
- Provider debug hook changes run focused hook Jest, TypeScript, resource/function map gates, the inventory gate, and target `git diff --check`.
- Debug harness replay changes run focused provider harness isolation Jest, TypeScript, resource/function/mainline map gates, the unified debug surface gate, the inventory gate, and target `git diff --check`.
- Map/doc changes run `npm run verify:resource-operation-map`, `npm run verify:function-map-compile-gate`, and `npm run verify:architecture-mainline-call-map` when owner bindings change.
- RSS improvement claims require release/global install, aggregate restart, health version alignment, and large-payload replay measurement. Without that, only source/native/build optimization is proven.

Read format / 阅读格式

图节点

人读标题 → 一句话职责 → contract node id。审计时先看职责是否在正确层，再看 raw node id。

图边

step_id → 语义动作。高风险边会写专用注释；普通边显示 caller module → callee module。

表格

表格保留真实 caller_symbol / callee_symbol / source file，用来回源代码验证，不让图承担所有细节。

Side-channel meaning / side-channel 含义

它不是资源

side-channel 是控制事实的传输方式，不是资源名、不是 payload 字段、也不是第二套 request/response。

真正资源/owner

这里的资源是 Error handler、provider health state、debug/error envelope 等明确 owner。HTML 的 Error resources 区块单独展示这些 owner/resource。

允许做法

主链节点可以把错误、health、debug 事实写给 Error handler / health owner；这些事实只影响错误投影、健康状态、路由可用性。

禁止做法

不得把 debug/error/metadata/control 字段塞进 provider request body 或 client response body；不得用 side-channel 重建业务 payload。

Error resources / 错误处理资源

错误链是独立资源链：错误进入 V3Error01SourceRaised 后逐节点分类、策略、执行决策、客户端投影。右侧两个 provider 节点不是错误来源：V3Transport13ResponsesHttpRequest 表示 provider send/transport 失败时写 health failure，V3ProviderResp14Raw 表示收到 raw 响应后写 health success；它们只服务 provider health/availability。

## L93-137 Stopless guidance is schema/help text, not task-state judgment

- Trigger: stopless / reasoningStop guidance, CLI summary, or continuation prompt tells the model the task is done, ready to close, must stop, should not continue, or otherwise decides terminal state for the model.
- Rule: keep production prompt text neutral. First no-schema continuation is exactly `继续。`; invalid-schema continuation is feedback repair such as `继续；按上一轮反馈修正。`; non-terminal CLI summary is `继续`; `stopreason=2` next-turn prompt is the model-provided `next_step` itself. Schema guidance may state field requiredness and numeric branch meanings only.
- Anti-pattern: hardcoded "suggested next step" prose, final-round wrap-up instructions, summary text that claims progress state, or any handler/SSE/outbound patch that tries to repair stopless behavior outside Rust Chat Process/servertool owners.
- Evidence target: Rust stopless/cli_contract tests, router-hotpath resume-guidance tests, stopless invalid-schema and contract blackboxes, `verify:servertool-rust-only`, global install/restart, installed `routecodex hook run reasoningStop` outputs, and runtime `dist` grep showing no terminal-state judgment wording.
